## Part A

This was an individual project so I did all of the work.
It feels good know that all of the work is mine and I was able to make something cool on my own.

This project really built up my research skills.
I had to research many academic papers to understand the math behind the Gaussian Splat rendering process.
I also had then apply the math in actual code to get things to work.

I had to overcome how extremely hard it is to know what is wrong in shader code.
If there is anything wrong in the code you render total nonsense, with little insight into what is wrong.
Which is to say that you can't just fix things till you have a solution, you have to just have the solution.
This means knowing exactly all of the complex math and knowing exactly how that translates into shader code.

## Part B

I accomplished an improved Gaussian Splat rendering plugin for Unity.
The plugin now correct occlusion between Gaussian Splat assets and isometric rendering.
These two features were the biggest items holding back the plugin.
There is no meaning to the visual accuracy of the technology if assets randomly pop in and out of each other.
Development used to be very painful without the isometric view, it would be impossible to line anything up correctly.
